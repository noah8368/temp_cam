int i3system.sdk.te.TE_A.recvImage ( short[]  buf,
boolean  agc 
)
inline

Receive image data.

Parameters
buf: buffer to get image data (384x288 for QVGA and 640x480 for VGA, 16bits)
agc: if set true, AGC algorithm is applied. Otherwise, AGC is not applied.
Returns
1: Read image successfully
2: Read image of size 0
3: Recieved image size is not equal to requested size
4: Data read fail
Remarks
This function receive data from Thermal Expert and calibrate to show image. It send image data to buf. If agc is set to true, gain and offset value is automatically calculated so that image data will be fitted to 16bit range.
Image values are inserted in signed buffer, but it is computed as unsigned value. Therefore, consider image data as unsigned value.

int i3system.sdk.te.TE_A.recvImage ( short[]  buf,

float  temp_max

Receive image data with temperature limit.

Parameters
buf: variable to get image data (384x288 for QVGA and 640x480 for VGA, 16bits)
temp_min: minimum temperature to express in image
temp_max: maximum temperature to express in image
Returns
1: Read image successfully
2: Read image of size 0
3: Recieved image size is not equal to requested size
4: Data read fail
Remarks
This function receive image data from Thermal Expert and manipulate data to express scene within temperature range. if temperature is smaller than temp_min, 0 will be assigned, and if temperature is greater than temp_max, 65535 will be assigned. temperature within those limit will set value between 0 and 65535 depending on temperature.
Image values are inserted in signed buffer, but it is computed as unsigned value. Therefore, consider image data as unsigned value.

◆ setEventHandler()

void i3system.sdk.te.TE_A.setEventHandler ( Handler  handler)
inline

Set handler to handle usb events.

Parameters
handler: handler to receive & handle usb events
Remarks
When usb state is changed, message related to usb is sent. This function set to handler to receive and handle events. 3 events are detected and sent to users.
MESSAGE_PERMISSION_GRANTED is sent when user allow permission to communicate. Since permission to communicate with usb device is required in android, this library check if permission is granted. If not, popup to ask user to allow communication will appear when calling openTE_A function, and if user allow this, MESSAGE_PERMISSION_GRANTED is sent to user.
MESSAGE_DEVICE_DETACHED is sent when device is detached from host device.
MESSAGE_DEVICE_ATTACHED is sent when device is attached to host device.
If user does not set handler, message will be discarded.

◆ setShutterMode()

boolean i3system.sdk.te.TE_A.setShutterMode ( short  mode)
inline

Set shutter operation mode.

Parameters
mode: Shutter operation mode. (Time mode = I3_TIME_SHUTTER, Temperature mode = I3_TEMP_SHUTTER, manual = I3_MANUAL_SHUTTER)
Returns
true : Command is transferred successfully
false : Transfer of command failed
Remarks
Shutter in TE engine is automatically closed and do calibration to maintain high image quality. This function set a operation mode of shutter. It supports time mode which closes shutter periodically with specified period and temperature mode which closes shutter when temperature of shutter is changed with specified amount.
If user don't want automatic calibration, set manual mode and call shutterCalibrationOn function to calibration when want to do calibration.
See SetShutterTemperature and SetShutterTime.
